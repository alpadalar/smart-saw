**File: src/gui/widgets/touch_button.py**

1. Create TouchButton class extending QPushButton
2. Enable touch events with `setAttribute(Qt.WA_AcceptTouchEvents, True)`
3. Override `event()` to handle QEvent.TouchBegin, TouchUpdate, TouchEnd, TouchCancel
4. On TouchBegin:
   - Check if touch point is within button bounds (strict bounds, no tolerance)
   - If first positioning button touch (track with class variable), set pressed state
   - Emit custom `touch_pressed` signal
   - Update visual state (setDown(True))
5. On TouchUpdate:
   - Check if touch point still within button bounds
   - If finger moved outside bounds: emit `touch_released`, setDown(False)
6. On TouchEnd/TouchCancel:
   - Emit `touch_released` signal
   - Update visual state (setDown(False))
7. Keep mouse events working via inherited behavior (DO NOT override mousePressEvent/mouseReleaseEvent)
8. Emit `pressed` and `released` signals for mouse compatibility (existing code)
9. Add custom signals: `touch_pressed = Signal()`, `touch_released = Signal()`

**File: src/gui/widgets/__init__.py**
- Create file if not exists
- Export TouchButton

**Key decisions from CONTEXT.md:**
- Instant activation (0ms) - no delay on touch down
- Strict button bounds - must touch exactly within button area
- Visual state maintained while finger on button
- Stop immediately if finger slides off button bounds
