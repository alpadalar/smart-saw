## Summary

Phase 24.1, v2.0 milestone auditinde tespit edilen iki config-kod uyumsuzlugunu ve bir dokuman hatasini kapatir. Hicbir yeni ozellik eklenmez; yalnizca mevcut davranis config-driven hale getirilir ve bir onay kutusu guncellenir.

**P1 — Eksik config bolumu:** `config.yaml` icerisinde `camera.vision` alt bolumu mevcut degil. `VisionService.__init__` bu bolumu `config.get("vision", {})` ile okumaya calisiyor; bulamadiginda `polling_interval=0.5s` ve `recording_duration=10.0s` sabit degerlerine dusuyor. Cozum: `config.yaml`'a `camera.vision` alt bolumunu eklemek.

**P2 — Olu config anahtarlari:** `config.yaml`'da `camera.health.broken_weight=0.70` ve `camera.health.wear_weight=0.30` tanimli. Ancak `HealthCalculator` bu degerleri hic okumuyor; `BROKEN_WEIGHT = 0.7` ve `WEAR_WEIGHT = 0.3` sinif sabitleri olarak hardcode edilmis. Rakamlar tesaduf esit oldugundan davranis bozulmamis, ancak config anlamsiz. Cozum: `HealthCalculator`'i constructor uzerinden config kabul edecek sekilde degistirmek ve `LDCWorker.run()` icinde instantiation sirasinda `camera_config` gecirilmesi.

**Dokuman hatasi:** `REQUIREMENTS.md` icerisinde GUI-06 satiri `[ ]` (tamamlanmamis) olarak isaretli; oysa `24-02-SUMMARY.md` audit pass oldugunu teyit ediyor ve kod `main_controller.py` line 213'te mevcut. Cozum: `[ ]` -> `[x]` degisikligi.

### P2 — Olu config anahtarlari (HealthCalculator)

**Audit kaniti:**
```
from: "config.yaml"
to: "HealthCalculator"
issue: "camera.health.broken_weight/wear_weight keys declared in config.yaml but
        never consumed. HealthCalculator uses hardcoded BROKEN_WEIGHT=0.7,
        WEAR_WEIGHT=0.3. Values match but config is dead."
severity: "P2"
```

**Dogrulanmis mevcut HealthCalculator (health_calculator.py):** [VERIFIED: dosya okuma]
```python
class HealthCalculator:
    BROKEN_WEIGHT = 0.7   # hardcode sinif sabiti
    WEAR_WEIGHT = 0.3     # hardcode sinif sabiti

    def calculate_saw_health(self, ...) -> float:
        broken_impact = (broken_pct / 100.0) * self.BROKEN_WEIGHT
        wear_impact = (normalized_wear / 100.0) * self.WEAR_WEIGHT
        ...
```

**Dogrulanmis config.yaml (satir 560-563):** [VERIFIED: dosya okuma]
```yaml
  health:
    broken_weight: 0.70    # var ama okunmuyor
    wear_weight: 0.30      # var ama okunmuyor
```

**Dogrulanmis LDCWorker instantiation (ldc_worker.py satir 80):** [VERIFIED: dosya okuma]
```python
health_calc = HealthCalculator()   # config gecilmiyor
```

**Gerekli degisiklik — iki adim:**

1. `HealthCalculator.__init__` ekle, constructor'dan config kabul etsin:
```python
def __init__(self, broken_weight: float = 0.7, wear_weight: float = 0.3) -> None:
    self.BROKEN_WEIGHT = broken_weight
    self.WEAR_WEIGHT = wear_weight
```

2. `LDCWorker.run()` icindeki instantiation'i guncelle:
```python
health_cfg = self._config.get("health", {})  # self._config = camera_config
health_calc = HealthCalculator(
    broken_weight=float(health_cfg.get("broken_weight", 0.7)),
    wear_weight=float(health_cfg.get("wear_weight", 0.3)),
)
```

**Gereksinim:** `LDCWorker.__init__`'in `config` parametresini saklamasi gerekiyor. Simdi kontrol ediliyor:

```python
# ldc_worker.py __init__ (satir 32-60) — config parametresi var ama alt alanlar extract edilip
# self._* degiskenlerine ataniyor; self._config olarak saklanmiyor.
```

Bu demek ki `self._config = config` atamasini ya eklemek gerekiyor, ya da `__init__` icinde `health_weight` degerlerini onceden cekip `self._broken_weight` / `self._wear_weight` olarak saklamak gerekiyor. **Ikinci yaklasim tercih edilir** — run() icinde heavy import oncesinde config okumasi daha temiz.

**Tercih edilen yaklasim:**
```python
# ldc_worker.py __init__ icerisine ekle:
health_cfg = config.get("health", {})
self._broken_weight: float = float(health_cfg.get("broken_weight", 0.7))
self._wear_weight: float = float(health_cfg.get("wear_weight", 0.3))

# ldc_worker.py run() icinde degistir:
health_calc = HealthCalculator(
    broken_weight=self._broken_weight,
    wear_weight=self._wear_weight,
)
```

## Architecture Patterns

### Config Okuma Paterni (mevcut proje standardi)

Proje boyunca kullanilan pattern: constructor'da `config.get("section", {})` ile alt bolum alinir, `section_cfg.get("key", default)` ile degerler cekilerek instance degiskenlerine atanir. [VERIFIED: vision_service.py, ldc_worker.py, camera_service.py incelendi]

```python
# Ornek: VisionService'in mevcut dogrru yaklasimi
vision_cfg = config.get("vision", {})
self._polling_interval: float = float(vision_cfg.get("polling_interval", 0.5))
```

`HealthCalculator` de ayni sekilde degistirilecek; `health_cfg = config.get("health", {})`.

### Sinif Sabiti -> Constructor Parametresi Pattern

```python
# Oncesi (hardcode)
class HealthCalculator:
    BROKEN_WEIGHT = 0.7
    WEAR_WEIGHT = 0.3

# Sonrasi (config-driven, geriye uyumlu default'lar)
class HealthCalculator:
    def __init__(self, broken_weight: float = 0.7, wear_weight: float = 0.3) -> None:
        self.BROKEN_WEIGHT = broken_weight
        self.WEAR_WEIGHT = wear_weight
```

Default degerler korundugundan `HealthCalculator()` cagrian testler ve diger kod kirilmaz.

## Common Pitfalls

### Pitfall 1: LDCWorker'da `self._config` saklamadan run() icinde config kullanmaya calismak
**Ne yanlis gider:** `run()` icinde `self._config` kullanilirsa `AttributeError`.
**Neden olur:** `__init__` config degerlerini alt alanlara parcaliyor, raw config'i saklamiyor.
**Nasil onlenir:** `__init__` icinde health weight degerlerini onceden `self._broken_weight` / `self._wear_weight` olarak cek. `run()` icinde bunlari kullan.

### Pitfall 2: `HealthCalculator` sinif sabitlerini instance sabitlerine cevirmek
**Ne yanlis gider:** `BROKEN_WEIGHT` ve `WEAR_WEIGHT` hala buyuk harfle yazilirsa "sabit" gorunumu verir ama artik instance degiskeni. Bu kafa karistirabilir.
**Nasil onlenir:** Instance degiskeni yapildiginda da buyuk harf isimlendirme kabul edilebilir (Python'da yaygın). Onemli olan `self.BROKEN_WEIGHT = broken_weight` atamasinin constructor'da yapilmasi.

### Pitfall 3: YAML indentation hatasi
**Ne yanlis gider:** `vision:` blogu yanlis indent duzeyine eklenirse `camera.vision` yerine `root.vision` olur ya da YAML parse hatasi olur.
**Nasil onlenir:** `health:` ile ayni indent duzeyinde, `camera:` altinda ekle. 2-space indent standardi.
