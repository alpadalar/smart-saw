<decisions>
## Implementation Decisions

### Close Button
- Icon: `src/gui/images/close.png`
- Size: 57×57 pixels
- Position: x=633, y=11 (within numpad dialog)
- Behavior: Clicking closes the numpad immediately **without sending any value** to the machine

### Numpad Pre-fill
- Applies to: cutting speed (kesme hızı) and descent/feed speed (ilerleme hızı) inputs
- On open: show current value instead of default `0`
- Once user types: normal input behavior continues (pre-filled value is replaced on first keystroke)
- No changes to other numpad uses (e.g., positioning, other parameters)

### Scope
- Changes confined to `src/gui/controllers/control_panel_controller.py` and/or the numpad widget class
- No backend, Modbus, ML, AI, or database changes
- No new dependencies

### Claude's Discretion
- Exact class/method names for numpad dialog (to be confirmed by reading existing code)
- Whether pre-fill replaces on first keystroke or clears on focus — standard numpad behavior: replace on first keystroke

</decisions>

<specifics>

</specifics>
