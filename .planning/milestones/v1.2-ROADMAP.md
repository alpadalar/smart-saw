# Milestone v1.2: ML Speed Memory & Chart UX

**Status:** SHIPPED 2026-01-15
**Phases:** 5-6
**Total Plans:** 2

## Overview

ML kesim sonrasi hiz restorasyonu ve grafik kullanilabilirligi iyilestirmesi. Operator hizlari ML kesim sirasinda korunur ve kesim sonrasi otomatik olarak geri yuklenir. Grafik eksenlerinde dinamik etiketler ile kullanici hangi metrigi goruntuledigini anlayabilir.

## Phases

### Phase 5: ML Speed Restoration

**Goal**: Save/restore kesme and inme speeds around ML cuts
**Depends on**: Phase 4 (Modbus Connection Timeout Handling)
**Plans**: 1 plan

Plans:
- [x] 05-01: Implement save/restore of cutting speeds around ML cuts

**Details:**
- Speed restore configuration added to config.yaml under ml.speed_restore section
- MLController saves original speeds when cutting starts (is_cutting becomes True)
- MLController restores original speeds when cutting ends via async _reset_cutting_state
- ModbusWriter wired from ControlManager to MLController for speed writes

**Files Modified:**
- config/config.yaml
- src/services/control/ml_controller.py
- src/services/control/manager.py

### Phase 6: Dynamic Chart Axis Labels

**Goal**: Add dynamic axis labels to sensor data charts
**Depends on**: Phase 5
**Plans**: 1 plan

Plans:
- [x] 06-01: Add dynamic Y-axis and X-axis title labels to cutting graph

**Details:**
- Y-axis title label positioned left of Y-axis value labels
- X-axis title label positioned below X-axis value labels
- Labels update dynamically when axis selection buttons are clicked
- Turkish characters display correctly (Kesme Hizi, Ilerleme Hizi, etc.)

**Files Modified:**

## Milestone Summary

**Key Decisions:**
- ModbusWriter created in ControlManager and passed to MLController via dependency injection
- Speed restoration is async to support Modbus write operations
- Feature is configurable via speed_restore.enabled and restore_on_cutting_end
- Used horizontal text for axis titles (Qt rotation is complex)
- Axis title style: Plus Jakarta Sans, bold, 16px, #F4F6FC
