# Phase 19.1 Plan 01: NumpadDialog Close Button and Pre-fill Summary

**Touch-UX improvement: NumpadDialog gains a dismiss close button and pre-fills current speed value with first-keystroke replacement for cutting and descent speed inputs.**

## Performance

- **Duration:** 1 min
- **Started:** 2026-03-25T21:19:41Z
- **Completed:** 2026-03-25T21:20:57Z
- **Tasks:** 2
- **Files modified:** 3 (numpad.py, control_panel_controller.py, close.png added)

## Accomplishments

- Added `closeButton` QPushButton at position (633, 11) sized 57x57 in `Ui_Dialog.setupUi()`, loading `close.png` with transparent/no-border style
- Added `initial_value` parameter to `NumpadDialog.__init__()` with `_prefilled` flag for first-keystroke replacement — digit press clears pre-fill, backspace trims normally
- Wired `closeButton.clicked` to `self.reject()` so dismiss sends no value to caller
- Updated both `_handle_cutting_speed_frame_click` and `_handle_descent_speed_frame_click` to compute `initial_str` before constructing `NumpadDialog` and pass it as `initial_value=initial_str`
- Removed stale "AFTER construction" and "showEvent reset" comments from both call-sites

## Task Commits

Each task was committed atomically:

1. **Task 1: Add close button and pre-fill support to NumpadDialog** - `cc10390` (feat)
2. **Task 2: Update call-sites in control_panel_controller.py to pass initial_str** - `355fb3a` (feat)

## Files Created/Modified

- `src/gui/numpad.py` - Added closeButton widget, initial_value parameter, _prefilled flag, close button connection, first-keystroke replacement in add_digit()
- `src/gui/controllers/control_panel_controller.py` - Updated cutting speed and descent speed call-sites to pass initial_value=initial_str to NumpadDialog
- `src/gui/images/close.png` - Close button icon (pre-existing file, now committed)

## Decisions Made

- `closeButton` uses `QPushButton` (not `QToolButton`) with transparent style — it is a dismiss action, not a numpad value button, so the gradient tool-button style would be misleading
- `backspace()` intentionally does NOT check `_prefilled` — trimming a pre-filled value (e.g., "120" → "12") is correct behavior per UX research
- `initial_value` defaults to `""` (empty string, not `None`) to keep `self.value` type-consistent as `str` throughout all methods
