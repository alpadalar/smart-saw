## Common Pitfalls

### Pitfall 1: Wrong geometry for btnSeritTork relocation

**What goes wrong:** The original `btnSeritTork.setGeometry(169, 239, 240, 69)` is called at line 719 during widget construction. If the relocation edit changes `y=239` to `y=238` (matching the CONTEXT.md spec of `y=238`), the 1-pixel difference is intentional and consistent with the new button's y position.

**Why it happens:** CONTEXT.md specifies `y=238` for both the relocated btnSeritTork and the new btnSeritGerginligi. The current btnSeritTork uses `y=239` (off-by-one). Use `y=238` for both per the spec.

**How to avoid:** Apply the geometry change directly in the `setGeometry` call at line 719. Change `(169, 239, 240, 69)` to `(41, 238, 240, 69)`.

**Warning signs:** If btnSeritTork appears 1px lower than btnSeritGerginligi after implementation, the y value was not updated.

### Pitfall 2: Button created after _setup_axis_button_groups() is called

**What goes wrong:** If `btnSeritGerginligi` is instantiated after `_setup_axis_button_groups()` runs, `getattr(self, 'btnSeritGerginligi', None)` returns `None` and the button is never added to `_y_group`, so the `_on_y_axis_changed` signal is never fired.

**Why it happens:** Construction order matters. All existing Y-axis buttons are created in a single `_setup_graph_widgets()` block before `_setup_axis_button_groups()` is called.

**How to avoid:** Create `self.btnSeritGerginligi` in the same block as the other Y-axis buttons (immediately after `btnSeritTork`), not in a separate method or later initialization step.

**Warning signs:** Clicking the new button does not update the graph.

### Pitfall 3: Forgetting to add the key to _get_selected_y_axis()

**What goes wrong:** The button toggles state visually and gets added to the group, but `_get_selected_y_axis()` falls through to the default `return "serit_kesme_hizi"`, so the graph always shows Kesme Hızı when the new button is selected.

**Why it happens:** Two separate code locations must be updated: the elif chain AND the y_titles dict. Easy to update one and miss the other.

**How to avoid:** The plan should list all 4 touch points as a checklist: (1) button creation + relocation, (2) _setup_axis_button_groups tuple, (3) _get_selected_y_axis elif, (4) update_axis_titles y_titles dict.

**Warning signs:** Graph does not change when new button is clicked, but button appears visually selected.
