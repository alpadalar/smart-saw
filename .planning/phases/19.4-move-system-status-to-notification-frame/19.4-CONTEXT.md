# Phase 19.4: Move System Status to Notification Frame - Context

**Gathered:** 2026-04-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Move the system status UI components (iconStatus and labelSystemStatusInfo) from the control panel page's dedicated SYSTEM STATUS FRAME into the always-visible notificationFrame at the top of the main window. Resize the log viewer to fill the freed space. No backend logic changes.

</domain>

<decisions>
## Implementation Decisions

### Which components to move
- Move `iconStatus` (QLabel with pixmap) from `systemStatusFrame` in `control_panel_controller.py` to `notificationFrame` in `main_controller.py`
- Move `labelSystemStatusInfo` (text label) from `systemStatusFrame` to `notificationFrame`
- Do NOT move the title label `labelSystemStatus` ("Sistem Durumu") — it becomes irrelevant once the frame is removed

### Target positions in notificationFrame
- `iconStatus`: size 35x35, position (560, 13) inside notificationFrame
- `labelSystemStatusInfo`: position (615, 18), left-aligned horizontally, vertically centered, width large enough to fit all possible status texts

### Original SYSTEM STATUS FRAME
- Remove `systemStatusFrame` entirely from `control_panel_controller.py`
- Remove all UI elements inside it: `labelSystemStatus`, `iconStatus`, `labelSystemStatusInfo`

### Log viewer resize (fills freed space)
- `logViewerFrame` geometry: from `(1176, 362, 321, 460)` → `(1176, 362, 321, 703)`
- `log_text` height: increase to match new frame height (currently 360, extend proportionally)

### Data flow / signal connections
- Keep all existing update logic intact — `labelSystemStatusInfo.setText(...)` and `iconStatus.setPixmap(...)` calls at lines ~1982, ~2152, ~2240 must continue working
- The labels are now parented to `notificationFrame` (owned by main_controller), so `control_panel_controller` needs a reference to them
- No new architecture, classes, or signals — pass references or access via parent; Claude's discretion on the exact mechanism as long as existing slots keep working

### Claude's Discretion
- Exact width of `labelSystemStatusInfo` in the notification bar (must fit all status texts)
- How `control_panel_controller` accesses `iconStatus` and `labelSystemStatusInfo` after they move to `main_controller` (direct reference pass, attribute on main, or similar — keep it simple)
- `log_text` exact height value (fill the new 703px frame cleanly)

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `notificationFrame` (main_controller.py:230): QFrame at (425, 38, 1465, 60), semi-transparent dark background, already has labelDate and labelTime as children
- `iconStatus` (control_panel_controller.py:910): QLabel with `setScaledContents(True)`, currently 71x71 — just re-parent and resize to 35x35
- `labelSystemStatusInfo` (control_panel_controller.py:921): QLabel with word wrap, center-aligned — re-parent, reposition, change alignment to left

### Established Patterns
- All labels in notificationFrame are QLabels created with the frame as parent and positioned with `setGeometry()`
- Status updates happen in `_update_system_status()` and related methods in control_panel_controller.py (~lines 1982, 2152, 2240)

### Integration Points
- `control_panel_controller.py` must retain `self.iconStatus` and `self.labelSystemStatusInfo` as attributes so existing update code (`self.iconStatus.setPixmap(...)`, `self.labelSystemStatusInfo.setText(...)`) continues to work
- `main_controller.py` creates `notificationFrame` and passes it (or its children) to `control_panel_controller` during `__init__`
