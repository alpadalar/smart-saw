# Phase 19.2: Graph Axis Labels - Context

**Gathered:** 2026-03-26
**Status:** Ready for planning
**Source:** User-provided inline requirements

<domain>
## Phase Boundary

Add two static QLabel widgets inside `kesimGrafigiFrame` that display the currently selected X and Y axis variable names on the cutting graph. Labels update dynamically when axis selection buttons are clicked. No graph rendering, backend, or data changes.

</domain>

<decisions>
## Implementation Decisions

### Y-Axis Label
- Parent: `kesimGrafigiFrame`
- Position: x=46, y=72
- Font size: 20pt
- Alignment: left
- Content: currently selected Y-axis variable name

### X-Axis Label
- Parent: `kesimGrafigiFrame`
- Position: x=806, y=518
- Font size: 20pt
- Alignment: right
- Content: currently selected X-axis variable name

### Widget Lifecycle
- Create labels once (in `__init__` or setup method) — do NOT recreate on each update
- Update text only via `setText()` when selection changes

### Update Trigger
- Labels update in the same code path where graph update is triggered after axis button clicks
- Axis selection is handled in existing button click handlers

### Scope
- Only `sensor_controller.py` is modified
- No changes to graph rendering logic
- No backend or data changes
- No `.ui` file changes — labels created programmatically

### Claude's Discretion
- Exact attribute names for the new label widgets (e.g., `self.yAxisLabel`, `self.xAxisLabel`)
- Where exactly in `sensor_controller.py` the labels are initialized (likely `__init__` or existing setup block)
- Initial text value before any selection is made

</decisions>

<specifics>
## Specific Ideas

- kesimGrafigiFrame is the parent widget for the cutting graph
- Labels sit on top of / alongside the graph, not inside the matplotlib canvas
- The button click handlers that update the graph are the place to also call `setText()`

</specifics>
